# 完成弹窗、片段列表与设备复选框视觉修复设计

## 1. 背景

多片段与可选设备功能完成后，Windows 实机截图暴露了三个局部视觉问题：

1. 转换完成弹窗横向过长，信息图标和正文之间存在大块空白；
2. 片段列表继承了系统深色列表背景，但文字仍使用应用的深蓝色，导致对比度不足；
3. 未勾选的兼容设备复选框在鼠标悬停时会与白色背景融为一体。

这些问题只影响界面表现，不影响转换结果、片段数据或点击行为。

## 2. 根因

- `QMessageBox QLabel` 当前带有 `min-width: 260px`。该规则不仅匹配正文标签，也匹配 Qt 内部的图标标签 `qt_msgboxex_icon_label`，因此图标列被错误撑宽。
- `QListWidget#segmentList` 没有显式背景、文字和选中状态样式。在 Windows 深色系统主题下，原生列表背景和应用全局文字色发生混用。
- `QCheckBox::indicator:unchecked:hover` 没有显式样式。Windows 原生悬停状态在当前混合主题下可能呈现白底或无边框，从而看似消失。

## 3. 方案选择

采用局部 QSS 修复，不新增自定义控件：

- `QMessageBox QLabel` 只负责文字颜色和透明背景；正文标签与图标标签分别取消强制最小宽度，让 Qt 根据内容计算紧凑布局。
- 为 `QListWidget#segmentList`、普通条目、悬停条目和选中条目设置浅色背景与深色文字。
- 只覆盖复选框的“未勾选”和“未勾选悬停”指示器；不覆盖 `checked` 状态，以保留当前用户认可的原生勾选效果和动效。

不采用以下方案：

- 不编写自定义弹窗、列表委托或复选框绘制代码，避免增加无障碍、键盘操作和高 DPI 适配负担；
- 不强制整个应用使用 Fusion 或固定浅色系统主题，避免无关控件外观发生变化；
- 不使用图片形式的复选框图标，避免丢失原生勾选动效。

## 4. 修改范围

- `livephoto/ui/theme.py`：新增或收窄三个控件的 QSS 规则；
- `tests/test_ui.py`：添加样式选择器回归测试，确保关键状态拥有明确的前景色、背景色和边框；
- `scripts/capture_ui.py`：不修改，只复用现有截图能力进行人工视觉核验。

## 5. 验收标准

1. 九个片段的完成提示保持两行文字，弹窗不再出现被图标列撑出的大片横向空白；
2. 片段列表在 Windows 深色和浅色系统主题下均为浅色列表、深色文字，普通、悬停和选中状态均清晰；
3. 未勾选设备的方框在普通与悬停状态下始终可见，悬停时使用蓝色边框反馈；
4. 已勾选状态与点击动效不被新的 QSS 规则覆盖；
5. 现有 UI 测试、全量测试和界面截图检查通过；
6. 只保留本地提交，不推送 GitHub。
